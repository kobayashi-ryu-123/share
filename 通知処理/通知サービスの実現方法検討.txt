ヒヤリハット発生 → 分析 → 一斉通知 の処理フロー

■理想
①走行中に車載機がヒヤリハットイベント検知
②分析サーバーへイベント情報を送信
③分析サーバーでヒヤリハットAI分析
④EventBridgeのイベント発行
⑤EventBridgeのが通知サービスを起動
⑥通知サービス（Lambdaなど）が一斉配信
・MQTT（AWS IoT Core）送信
・FCM（Firebase Cloud Messaging）送信
⑦タブレット端末が受信
・アプリ起動中	→MQTT受信して、即時表示
・バックグラウンド	→FCM受信して、通知表示
・スリンガー中	→FCM受信して、復帰後に確認可能
・復帰時		→未取得イベント同期

| 項目			| MQTT   		| FCM			|
| -------------		| ---------		| --------- 		|
| リアルタイム性		| ◎（最速）     	| ○（数秒遅延あり）	|
| バックグラウンド		| ×（基本届かない）	| ◎（届く）		|
| スリープ中		| ×         			| ◎				|
| Angular Web対応	| ○（接続中のみ）  	| ◎（PWA可）	|
| 電池効率		| △         		| ◎				|
| 実装難易度		| やや高       		| 低				|


■最低限
①～③は同じ
④分析サーバが通知APIを起動
⑤通知APIがWebSochetへブロードキャスト
⑥Webアプリ接続中のタブレット端末が受信
